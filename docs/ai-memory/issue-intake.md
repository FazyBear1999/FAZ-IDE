# Issue Intake

## Purpose
- Structured issue requests for AI sessions.
- Keeps bug-fix prompts complete and reproducible.

## Workflow
- Add entries with `npm run ai:issue -- "<short summary>"`.
- Fill repro, expected/actual, and acceptance-test fields before handing to AI.

## Entry Template
- Summary
- Severity (low/medium/high)
- Repro Steps
- Expected
- Actual
- Suspected Area
- Options Considered (quick fix vs root fix)
- Assumptions (verified/partial/unverified)
- Acceptance Test
- Rollback Trigger
- Notes
